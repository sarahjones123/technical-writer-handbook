# Technical writer handbook

Welcome to the **Technical Writer Handbook**.

This handbook provides practical guidance for creating clear, consistent, and high-quality technical documentation.

It serves as a central reference for writing standards, documentation quality, and Docs-as-Code practices.

## Purpose

This handbook helps you:

- Write clear and concise documentation.
- Follow industry-standard writing practices.
- Maintain consistency across documentation.
- Improve documentation quality using automation.
- Continuously improve your technical writing skills.

## Intended audience

This handbook is intended for:

- Technical writers
- Documentation leads
- Product managers
- Software engineers who contribute to documentation
- Documentation reviewers

## Writing standard

This handbook adopts the **Google Developer Documentation Style Guide** as the primary writing standard.

As the handbook evolves, additional guidance and organization-specific standards may be added where appropriate.

## Handbook structure

The handbook currently includes the following sections:

- Getting started
- Writing standards
- Documentation quality
- Glossary

## Guiding principles

The handbook is based on the following principles:

- Write for the reader.
- Keep documentation clear and concise.
- Be consistent.
- Treat documentation as code.
- Automate quality checks whenever possible.
- Continuously improve documentation.

## Next steps

If you're new to this handbook, begin with **Getting started** and then continue to **Writing standards** before exploring the remaining sections.
