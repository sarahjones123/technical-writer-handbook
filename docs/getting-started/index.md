# Getting started

## Overview

Welcome to the Technical Writer Handbook.

This handbook provides practical guidance for writing and maintaining technical documentation using industry best practices.

It is designed to be used as a day-to-day reference rather than a book.

## Before you begin

Familiarize yourself with:

- Markdown
- Git
- MkDocs
- Google Developer Documentation Style Guide

## Guiding principles

- Write for the reader.
- Keep documentation simple.
- Be consistent.
- Prefer clarity over cleverness.
- Continuously improve documentation.
