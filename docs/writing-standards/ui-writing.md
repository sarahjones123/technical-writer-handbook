# UI writing

## Overview

Use consistent terminology when referring to user interface (UI) elements. Clearly distinguish UI labels from regular text.

## Guidelines

Follow these guidelines when writing about UI elements:

- Format UI elements using backticks.
- Use the exact UI label.
- Use sentence case unless the UI uses a different style.
- Use the imperative mood for instructions.
- Avoid unnecessary words.

## Best practices

### Buttons

**Instead of**

> Click Save.

**Write**

> Click `Save`.

### Menu paths

```text
Go to `Administration > Users`.
```

### Fields

```text
Enter the customer name in `Customer Name`.
```

### Check boxes

```text
Select the `Active` checkbox.
```

### Dialog boxes

```text
Click `OK`.
```

## Checklist

Before publishing documentation, verify that:

- UI labels use backticks.
- UI labels match the application.
- Instructions use the imperative mood.
- Menu paths are accurate.
