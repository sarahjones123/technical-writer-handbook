# Headings

## Overview

Headings organize content and help readers quickly find the information they need. Use clear, descriptive headings to improve readability and navigation.

## Guidelines

Follow these guidelines when writing headings:

- Use sentence case.
- Keep headings concise and descriptive.
- Organize headings in a logical hierarchy.
- Avoid skipping heading levels.
- Use headings that describe the content that follows.
- Avoid vague or generic headings.

## Best practices

### Use sentence case

Capitalize only the first word and proper nouns.

**Instead of**

```text
Configure Billing Rules
```

**Write**

```text
Configure billing rules
```

### Write descriptive headings

Choose headings that clearly describe the content.

**Instead of**

```text
Configuration
```

**Write**

```text
Configure billing rules
```

### Keep headings concise

Keep headings short and easy to scan.

**Instead of**

```text
Steps to configure the billing rules for an LDC
```

**Write**

```text
Configure billing rules
```

### Use a logical hierarchy

Organize headings in a consistent structure.

```text
# Configure billing rules

## Create a billing rule

## Edit a billing rule

## Delete a billing rule
```

### Avoid skipping heading levels

Maintain a logical heading hierarchy.

**Instead of**

```text
# Configure billing rules

### Create a billing rule
```

**Write**

```text
# Configure billing rules

## Create a billing rule
```

## Checklist

Before publishing documentation, verify that:

- Headings use sentence case.
- Headings clearly describe the content.
- Headings are concise.
- Heading levels follow a logical hierarchy.
- Heading levels are not skipped.
