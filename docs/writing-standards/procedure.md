# Procedures

## Overview

Procedures help users complete a task by providing clear, step-by-step instructions.

Well-written procedures are easy to follow, concise, and action-oriented.

## Guidelines

Follow these guidelines when writing procedures:

- Use numbered lists for sequential steps.
- Begin each step with an action verb.
- Describe one action per step.
- Write in the imperative mood.
- Present steps in the order they are performed.
- Include additional information only when it helps the reader complete the task.

## Best practices

### Use numbered lists

Use numbered lists for all sequential procedures.

**Example**

1. Open the application.
2. Select `Administration`.
3. Click `Users`.
4. Click `Add User`.

### Start each step with an action

Begin each step with an action verb.

**Instead of**

> The user should click `Save`.

**Write**

> Click `Save`.

### Describe one action per step

Each step should describe a single action.

**Instead of**

> Open the application, configure the settings, and save the changes.

**Write**

1. Open the application.
2. Configure the settings.
3. Click `Save`.

### Keep steps concise

Keep instructions short and easy to follow.

**Instead of**

> Navigate to the Administration page where you can configure user settings for the application.

**Write**

> Go to `Administration`.

### Provide context when necessary

Add a note only when additional information helps the reader complete the task.

> **Note**
>
> Administrator privileges are required to perform this procedure.

## Checklist

Before publishing a procedure, verify that:

- Steps are presented in the correct order.
- Each step begins with an action verb.
- Each step describes one action.
- Procedures use numbered lists.
- UI elements are formatted using backticks.
- Additional information is included only when necessary.
