# Voice and tone

## Overview

The voice and tone of your documentation determine how readers experience your content. Effective technical documentation is clear, direct, professional, and focused on helping users accomplish their goals.

This handbook adopts the Google Developer Documentation Style Guide as the primary writing standard.

## Guidelines

Follow these guidelines when writing documentation:

- Prefer active voice.
- Write in the present tense.
- Use the imperative mood for procedures.
- Write for the reader.
- Keep sentences concise.
- Use simple and familiar language.
- Avoid unnecessary jargon.
- Be consistent in terminology and style.

## Best practices

### Use active voice

Prefer active voice whenever it improves clarity.

**Instead of**

> Customer records are managed by the application.

**Write**

> The application manages customer records.

### Use the present tense

Describe how the product works today.

**Instead of**

> The application will display the report.

**Write**

> The application displays the report.

### Use the imperative mood

Write procedural steps as direct instructions.

**Instead of**

> The Save button should be clicked.

**Write**

> Click `Save`.

### Write for the reader

Focus on what the reader wants to accomplish.

**Instead of**

> The Billing page provides functionality for configuring billing rules.

**Write**

> Configure billing rules on the `Billing` page.

### Keep sentences concise

Express one idea per sentence whenever possible.

## Examples

| Avoid                                            | Preferred                                 |
| ------------------------------------------------ | ----------------------------------------- |
| The Save button should be clicked.               | Click `Save`.                             |
| The application will generate the report.        | The application generates the report.     |
| Customer records are managed by the application. | The application manages customer records. |
| Click on `Save`.                                 | Click `Save`.                             |

## Checklist

Before publishing documentation, verify that:

- Active voice is used where appropriate.
- The present tense is used consistently.
- Procedures use the imperative mood.
- Sentences are concise and easy to read.
- Terminology is consistent throughout the document.
