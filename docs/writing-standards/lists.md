# Lists

## Overview

Lists make documentation easier to read and scan. Use lists to present related information or sequential tasks.

## Guidelines

Follow these guidelines when writing lists:

- Use numbered lists for sequential tasks.
- Use bullet lists for related information.
- Keep list items concise.
- Write list items in a parallel structure.
- Keep punctuation consistent.

## Best practices

### Use numbered lists for procedures

```text
1. Open the application.
2. Select `Administration`.
3. Click `Save`.
```

### Use bullet lists for related information

```text
- Customer information
- Billing information
- Contact information
```

### Use parallel structure

**Instead of**

```text
- Configure billing rules
- Customer management
- Creating reports
```

**Write**

```text
- Configure billing rules
- Manage customers
- Generate reports
```

## Checklist

Before publishing documentation, verify that:

- The correct list type is used.
- List items are concise.
- List items follow a parallel structure.
- Punctuation is consistent.
